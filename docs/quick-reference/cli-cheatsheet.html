thai-lint CLI Cheatsheet

← Back to Quick Reference

Command Syntax

thailint [GLOBAL_OPTIONS] COMMAND [OPTIONS] [PATH...]

All commands accept multiple paths (files or directories). Defaults to current directory if no path specified.

Global Options

--version Show version and exit
--verbose, -v Enable debug output
--config, -c PATH Specify config file path
--project-root PATH Explicit project root (Docker/monorepo)
--help Show help message

Linting Commands

Command Purpose Key Options
nesting Detect excessive nesting depth --max-depth N, --parallel
srp Single Responsibility Principle violations --max-methods N, --max-loc N, --parallel
dry Detect duplicate code (DRY violations) --min-lines N, --min-occurrences N
magic-numbers Detect unnamed numeric literals --allowed-numbers LIST
file-placement Check file organization rules --recursive/--no-recursive
file-header Validate file headers --recursive/--no-recursive
print-statements Detect print/console.log statements --recursive/--no-recursive
method-property Methods that should be @property --recursive/--no-recursive
stateless-class Classes without instance state --min-methods N
pipeline For loops with embedded filtering --min-continues N

Common Command Options

--format, -f text | json | sarif
--recursive Scan directories recursively (default)
--no-recursive Only scan top-level files
--parallel, -p Enable multi-core processing
Tip: Use --format sarif for CI/CD integration with GitHub Code Scanning.

Output Formats

text Human-readable console output (default)
json Machine-readable JSON array
sarif SARIF v2.1.0 for GitHub/CI integration

Exit Codes

0 Success - no violations found
1 Violations found
2 Error (invalid args, config, etc.)

Quick Examples

# Lint current directory thailint nesting .
# Multiple paths with JSON output thailint srp -f json src/ lib/
# SARIF for GitHub Actions thailint magic-numbers -f sarif src/ > results.sarif
# Parallel processing for large repos thailint nesting --parallel src/
# Docker with explicit project root docker run -v $(pwd):/code thailint \ --project-root /code nesting /code/src/

Configuration Commands

init-config Create .thailint.yaml with defaults
config show Display current configuration
config get KEY Get specific config value
config set KEY VAL Set config value
config reset Reset to defaults
